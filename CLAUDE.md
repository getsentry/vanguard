This file provides guidance to Claude Code (claude.ai/code) and other AI coding agents when working with code in this repository.

## Project Overview

Vanguard is an internal blog-like platform inspired by PlanetScale's Beam. It's designed for creating permanence around timely internal moments — posts, categories, comments, reactions, and feeds — with Google OAuth (workspace-domain restricted) and basic email/password as fallback, plus role-based access (`admin`, `canPostRestricted`).

## Tech Stack

Vanguard runs on a Vercel-native stack:

- **Framework:** [React Router v7](https://reactrouter.com/) in framework mode (Remix's official successor, Vite-native)
- **Build pipeline:** Vite, driven via [`vp`](https://vite.plus/) (Vite Plus) or `pnpm`
- **ORM:** [Drizzle](https://orm.drizzle.team/) with the `drizzle-orm/node-postgres` driver (works with both local Postgres and Neon)
- **Database:** [Neon](https://neon.tech/) Postgres (local Postgres in Docker for dev/tests)
- **File storage:** [Vercel Blob](https://vercel.com/docs/storage/vercel-blob) with a local filesystem fallback for offline dev
- **Hosting:** Vercel (Preview Deployments wired to Neon branches)
- **Auth:** [`remix-auth`](https://github.com/sergiodxa/remix-auth) (still works on RR7) with a custom `GoogleStrategy` supporting the `hd` workspace-domain parameter
- **Observability:** `@sentry/react-router` for server + client, sourcemaps uploaded by `@sentry/vite-plugin`
- **Background work:** `waitUntil()` from `@vercel/functions` (with a local-dev polyfill) for fire-and-forget email / Slack notifications
- **Testing:** Vitest + happy-dom, running against a real local Postgres (never mocked)
- **Package manager:** pnpm (enforced) · **Node:** 20.x (pinned)

### Routing conventions (RR7 — same as Remix)

- **Dots nest routes:** `admin.categories._index.tsx` → `/admin/categories`
- **`$param` for dynamic segments:** `p.$postId._index.tsx` → `/p/:postId`
- **Trailing `_` opts OUT of parent layout:** `p.$postId_.edit.tsx` is NOT nested under `p.$postId`
- **Leading `_` = layout-only (no segment):** `_index.tsx` → `/`, `admin.tsx` is the admin layout
- **`$` catch-all:** `image-uploads.$.tsx` → `/image-uploads/*`
- **`[.]` for literal dots in URLs:** `feeds.$feedId[.]xml.tsx` → `/feeds/:feedId.xml`

### Data layer

Every data access helper ends in `.server.ts` and lives under `app/models/`. They import `db` from `~/db/client` and operators from `drizzle-orm`:

```typescript
import { eq, and, ilike, sql } from "drizzle-orm";
import { db } from "~/db/client";
import { posts, users } from "~/db/schema";
```

Patterns:

- **Reads with relations:** `db.query.posts.findFirst({ with: { author: true, category: true } })`
- **Reads with filters:** `db.select().from(posts).where(and(eq(posts.deleted, false), ...))`
- **Writes:** `db.insert(users).values({...}).returning()` — ALWAYS use `.returning()` to get inserted rows back
- **Upserts:** `.onConflictDoUpdate({ target: users.externalId, set: { email } })`
- **Transactions:** `db.transaction(async (tx) => { ... })` — use `tx` inside, NOT `db`
- **Full-text search:** raw SQL via `sql\`to_tsvector('english', ${posts.title}) @@ plainto_tsquery('english', ${query})\``
- **Types:** export inferred types from the schema: `export type User = typeof users.$inferSelect;` — DO NOT import types from anywhere Prisma-shaped.

### Auth (MANDATORY rules)

Authentication is enforced per-route. **Every protected route MUST enforce auth in BOTH the loader AND the action**, and **must have a test asserting that enforcement happens**. This is the #1 rule of this codebase.

```typescript
import type { LoaderFunctionArgs, ActionFunctionArgs } from "react-router";
import { requireUserId } from "~/services/auth.server";

export async function loader({ request }: LoaderFunctionArgs) {
  const userId = await requireUserId(request);
  // ...
}

export async function action({ request }: ActionFunctionArgs) {
  await requireUserId(request);
  // ...
}
```

Guard helpers in `app/services/auth.server.ts`:

- `getUserId(request)` → `string | undefined`
- `getUser(request)` → `User | undefined`
- `requireUserId(request)` → `string` (throws redirect to `/login?redirectTo=...`)
- `requireUser(request)` → `User`
- `requireAdmin(request)` → `User` (throws redirect to `/403` if not admin)

**No `context` parameter.** RR7 without a custom server has no `getLoadContext()` — guards parse the session cookie directly from the request.

### Loader / action return shapes

- **Success:** return a plain object. No `json()` helper. `return { posts, user };`
- **Errors with status:** `return Response.json({ error: "Bad request" }, { status: 400 });` or `return data({ error }, { status: 400 })` from `react-router` (use `data()` when you need headers alongside).
- **Redirects:** `throw redirect("/login")` or `return redirect("/foo")` — from `react-router`.

### Config injection (client + server)

- **Server-only env:** `process.env.FOO` — anywhere in `*.server.ts` or loaders/actions.
- **Client-visible env:** must be prefixed `VITE_` and read via `import.meta.env.VITE_FOO` — no more `window.CONFIG` injection.
- `app/config.tsx` centralises the client-side config object.

### File uploads

Use `request.formData()` (Web Standard, built into RR7) — never `unstable_parseMultipartFormData`. Call `uploadFile()` from `app/lib/upload-handler.ts`:

```typescript
export async function action({ request }: ActionFunctionArgs) {
  await requireUserId(request);
  const formData = await request.formData();
  const file = formData.get("file");
  if (!(file instanceof File))
    return Response.json({ error: "no file" }, { status: 400 });
  const { url } = await uploadFile({
    filename: file.name,
    buffer: Buffer.from(await file.arrayBuffer()),
  });
  return { url };
}
```

`uploadFile()` picks Vercel Blob if `BLOB_READ_WRITE_TOKEN` is set, otherwise falls back to `os.tmpdir()` + the `image-uploads.$` serve route.

### Fire-and-forget work (email / Slack)

Serverless functions terminate at response send, which kills un-awaited promises. Wrap background work with the local `waitUntil` helper:

```typescript
import { waitUntil } from "~/lib/wait-until";

export async function announcePost(post: PostQueryType) {
  waitUntil(
    (async () => {
      await Promise.all(emails.map((e) => email.notify({ post, config: e })));
      await Promise.all(slacks.map((s) => slack.notify({ post, config: s })));
    })(),
  );
}
```

The helper calls Vercel's `waitUntil` in production; in local dev, it just lets the promise run with a console-level error handler.

## Testing

- Vitest + happy-dom, real Postgres (no mocking of the DB)
- `test/setup-test-env.ts` truncates all tables before each test and reseeds `DefaultFixtures.DEFAULT_USER`
- Test fixtures in `app/lib/test/fixtures.ts` use faker + a cuid suffix to guarantee uniqueness (don't remove the suffix — it prevents flaky collisions on `Category.slug`, `Category.name`, `Feed.name`)
- Route tests live as siblings: `app/routes/p.$postId._index.test.ts` next to `app/routes/p.$postId._index.tsx`
- Every auth-protected endpoint must have a test that hits it without a session and asserts the redirect

**Most route test files carry `// @ts-nocheck`** because RR7's `LoaderFunctionArgs` type requires `unstable_url` and `unstable_pattern` fields that test argument builders don't currently populate. This is an accepted trade-off; a proper test helper that builds compliant `LoaderFunctionArgs` would be a future cleanup.

## Deployment

Deploys run on Vercel. The repo is wired to the Vercel project via the Vercel GitHub integration:

1. Production deploys: merges to `main` → Vercel production.
2. Preview deploys: every push to a feature branch → Vercel preview with its own Neon branch.
3. Build command: `pnpm build` (runs `react-router build`).
4. Output: Vercel React Router preset consumes `build/client` + `build/server`.
5. Sourcemaps auto-upload to Sentry when `SENTRY_AUTH_TOKEN` + `SENTRY_ORG` + `SENTRY_PROJECT` are set (via `@sentry/vite-plugin`).

Vercel dashboard: https://vercel.com/sentry/vanguard

## Things NOT to do

- ❌ Don't use npm or yarn — pnpm is enforced by the `preinstall` hook.
- ❌ Don't import from `@prisma/client` — it's gone. Types come from Drizzle schema (`typeof table.$inferSelect`).
- ❌ Don't import from `@remix-run/*` — use `react-router`.
- ❌ Don't use `json()` for responses — return plain objects, or `Response.json(body, { status })` / `data(body, { status })` for non-200.
- ❌ Don't use `unstable_parseMultipartFormData` — use `request.formData()`.
- ❌ Don't put `await requireUserId(request, context)` (two-arg form) — the `context` is gone.
- ❌ Don't commit `.react-router/` — it's generated and gitignored.
- ❌ Don't add `@ts-ignore` or `@ts-expect-error` to production code without a clear comment explaining the library-level issue being suppressed. Test files may use `@ts-nocheck` for RR7 stub-arg friction.
- ❌ Don't introduce background work without `waitUntil` — un-awaited promises die when the response sends.
